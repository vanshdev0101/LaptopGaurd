HOW IT WORKS

One failed login triggers a five-step chain.

Every incident moves through the same pipeline, start to finish, in under a second.

01 — DETECT
A failed sign-in hits the Windows Security Log

EventMonitor.cs polls for Event ID 4625 — the code Windows logs the moment a sign-in fails, whether it's a typo or someone who shouldn't be there.

02 — CAPTURE
The webcam, the USB ports, and every running process, all at once

CameraCapture.cs fires the webcam immediately. UsbMonitor.cs checks for devices connected in the last few seconds. AppMonitor.cs snapshots every running process.

03 — CORRELATE
Everything gets stitched into one incident

The service ties the photo, the process list, and any nearby USB activity together into a single record — no separate logs to cross-reference later.

04 — STORE
Written to disk, photos encrypted

Database.cs writes the incident to a local SQLite file. Evidence photos are encrypted at rest with AES-256.

05 — SURFACE
Live on the dashboard, wherever you are

LaptopGuard.Web exposes it through a TOTP-protected API. The dashboard renders it live — reachable only over your private Tailscale network, never the public internet.

SECURITY MODEL

The dashboard is only as safe as its own access.

Watching for intruders means the watcher itself can't be the weak point.

AUTH

TOTP sign-in

Compatible with Google Authenticator and Authy — the same 6-digit code standard, no separate app to install.

SESSIONS

8-hour tokens, in memory

Session tokens live in the backend's memory only. Restart the service and every session ends.

NETWORK

Tailscale-only exposure

The API is never reachable from the public internet — only from devices on your private Tailscale network.

STORAGE

AES-256 photo encryption

Evidence photos are encrypted at rest, so the image files themselves are useless without the key.

UNDER THE HOOD

Built with a small, deliberate stack.

No framework was reached for out of habit — each piece was picked for what this project needed to learn.
